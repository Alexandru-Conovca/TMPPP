# TMPPP — учебная «Система умного дома»

Проект демонстрирует классические паттерны на Python без сторонних библиотек:

Используемые паттерны:
 Порождающие (Creational)
 Factory Method – создание различных устройств (лампы, термостаты, камеры).
 Abstract Factory – фабрики для разных брендов устройств (Xiaomi, Philips, Samsung).
 Singleton – менеджер состояний всех устройств в доме.
 Builder – создание сценариев автоматизации (включение света по расписанию, настройка сценариев “утро/вечер/отсутствие дома”).
 Prototype – клонирование настроек для разных комнат (одинаковые сценарии, устройства с похожими настройками).

 Структурные (Structural)
 Adapter – поддержка устройств с разными API (Zigbee, Wi-Fi, Bluetooth).
 Decorator – расширение функций устройств (интеллектуальный свет, динамическая температура, энергосбережение).
 Facade – единый интерфейс для управления всеми устройствами дома (“всё включить/выключить” одним вызовом).
 Composite – групповое управление устройствами (все лампы на этаже, все датчики безопасности).
 Proxy – контроль доступа к устройствам или API: логирование команд, кэширование состояния, ограничения.

 Поведенческие (Behavioral)
 Observer – система уведомлений при событиях: движение на камере, изменение температуры, открытие двери.
 Strategy – выбор алгоритма энергосбережения: автоматический, пользовательский, ночной режим.
 Command – инкапсуляция действий: “включить лампу”, “выключить кондиционер”, “отправить уведомление”.
 Template Method – шаблон сценария автоматизации: последовательность действий “утро/вечер/отсутствие дома”.
 Chain of Responsibility – обработка событий от датчиков: система проверяет приоритеты и решает, что выполнить.
